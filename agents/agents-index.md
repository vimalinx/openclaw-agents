## 如何调用子代理

### 主代理（Wilson）职责

1. 分析任务类型
2. 选择合适的子代理
3. 编写清晰的任务描述
4. 委派任务给子代理
5. 监控执行进度
6. 审核结果质量

### 子代理响应原则

**子代理职责**:
1. 理解任务描述
2. 选择合适工具执行
3. 完成后通知主代理
4. 提供结果和报告
5. 不超出任务范围

---

## 任务委派流程

```
用户需求
    ↓
主代理分析（Wilson）
    ↓
选择子代理（XHS Operator / Boss Assistant）
    ↓
编写任务描述
    ↓
委派任务
    ↓
子代理选择工具
    ↓
子代理执行
    ↓
完成通知
    ↓
主代理审核
    ↓
返回结果给用户
```

## 质量标准

### 主代理职责

- **任务描述清晰**: 子代理能准确理解需求
- **代理选择正确**: 选择最合适的子代理
- **结果审核严格**: 检查输出质量和完整性
- **反馈及时**: 及时给用户反馈

### 子代理职责

- **任务执行完整**: 完成所有要求的任务
- **工具选择正确**: 选择最合适的工具
- **输出格式正确**: 按要求的格式输出
- **质量保证**: 输出符合专业标准
- **问题报告**: 及时报告发现的问题

## 协作原则

### 主代理

1. **不越权** - 不做子代理擅长的工作
2. **善协调** - 协调两个子代理的工作（如需要）
3. **快响应** - 保持对话流，不阻塞用户
4. **重质量** - 严格审核子代理输出

### 子代理

1. **专注执行** - 专注于分配的任务
2. **及时沟通** - 发现问题及时沟通
3. **质量第一** - 不为了速度牺牲质量
4. **主动反馈** - 发现潜在问题主动反馈

---

## 冲突解决

### 主代理决策优先

当两个子代理有分歧时：
1. 主代理根据目标做决策
2. 向子代理解释决策理由
3. 记录决策过程

### 子代理如何处理

当任务不明确时：
1. 不随意猜测需求
2. 及时向主代理确认
3. 提供多个选项供选择

## 子代理能力边界

### XHS Operator 能做什么

✅ **可以做的**:
- 小红书内容策划和优化
- 数据分析和趋势追踪
- 竞品监控和分析
- 运营策略制定
- 使用数据驱动系统（search.py, reasoning.py）
- 自动发布内容
- 热点追踪和爆款分析
- 生成各种报告（数据、竞品、策略）

❌ **不能做的**:
- 日程管理和会议协调
- 邮件处理和任务管理
- 跨系统架构设计
- 代码开发和系统集成

### Boss Assistant 能做什么

✅ **可以做的**:
- 日程管理和提醒
- 邮件汇总和分类
- 任务清单管理
- 会议协调和记录
- 信息整理和报告生成
- HEARTBEAT汇报
- 使用所有生产力工具（飞书、Excel等）
- AI辅助撰写和整理
- 生成各种报告（周报、月报、商业报告）
- 文档管理和归档

❌ **不能做的**:
- 小红书具体运营执行（交给 XHS Operator）
- 代码开发和系统集成
- 深度数据分析和建模
