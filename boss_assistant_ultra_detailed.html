📊 **数据驱动决策**：基于多维度数据（热点、竞品、用户行为）提供运营建议 - 🤖 **智能任务分配**：自动识别优先级，智能分配资源 - ⚡ **全流程自动化**：从热点监控到内容发布全自动化 - 📈 **实时监控反馈**：持续跟踪效果，自动优化策略 - 🎯 **个性化配置**：支持多账号、多平台、多策略配置 ### 1.2 适用场景 - 🎨 **内容创作者**：自媒体运营、小红书矩阵、视频制作 - 🏢 **商业运营**：商场运营、客户管理、数据分析 - 💼 **项目管理**：任务追踪、进度监控、团队协作 - 📊 **数据分析**：趋势分析、竞品监控、用户画像 ### 1.3 系统特性 **成熟特性**： - ✅ **模块化架构**：5 大功能模块，每个模块独立可测试 - ✅ **配置驱动**：所有行为通过配置文件控制，无需修改代码 - ✅ **日志记录**：完整的操作日志，支持调试和审计 - ✅ **错误处理**：完善的异常处理和错误恢复机制 - ✅ **性能优化**：缓存、异步处理、批量操作优化 - ✅ **可扩展性**：支持插件式扩展，可自定义新功能 --- ## 二、技术架构 ### 2.1 系统架构图 ``` ┌─────────────────────────────────────────────────────────────┐ │ Wilson 老板助理 │ ├─────────────────────────────────────────────────────────────┤ │ │ │ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ 信息获取层 │ │ 决策支持层 │ │ 执行自动化层 │ │ └──────────────┘ └──────────────┘ └──────────────┘ │ │ │ │ │ │ │ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ HEARTBEAT │ │ 任务管理 │ │ 小红书闭环 │ │ │ 邮箱检查 │ │ 日程管理 │ │ 热点监控 │ │ │ 日程提醒 │ │ 优先级 │ │ 内容生成 │ │ │ 项目进度 │ │ 任务统计 │ │ 配图生成 │ │ │ 热点追踪 │ │ 任务归档 │ │ 自动发布 │ │ │ 竞品监控 │ │ │ │ 数据反馈 │ │ │ 机会推荐 │ │ │ │ │ │ └──────────────┘ └──────────────┘ └──────────────┘ │ │ │ │ ┌──────────────────────────────────┐ │ │ │ 核心数据层 │ │ │ │ - IMAP 服务器地址（待实现） - 邮箱账号和密码（待实现） - 重要邮件过滤规则（待实现） #### 模块 1.2：日程提醒 **功能说明**：查看即将到来的会议和事件，提前准备和提醒 **实现原理**： 1. 读取 `calendar.json` 中的日程数据 2. 计算当前时间与日程时间的差值 3. 筛选出 24 小时内的日程 4. 提前 2 小时（可配置）发送提醒 **使用方法**： ```bash # 查看即将到来的日程（24小时内） python3 calendar-manager.py upcoming # 添加日程 python3 calendar-manager.py add "客户会议" 2026-02-21 14:00 # 查看指定日期的日程 python3 calendar-manager.py list 2026-02-21 ``` **配置选项**： - 提前提醒时间（默认 2 小时） - 日程时间格式（YYYY-MM-DD HH:MM） - 日程类型（会议、截止日期、提醒） #### 模块 1.3：热点追踪 **功能说明**：监控小红书目标关键词的热度变化，发现潜在爆款话题 **实现原理**： 1. 定义监控关键词列表（如："AI工具"、"效率神器"、"副业搞钱"） 2. 使用 MediaCrawler 定期爬取相关数据 3. 分析笔记数量、互动率、发布频率等指标 4. 识别热度上升的趋势，标记为潜在爆款 **使用方法**： ```bash # 启动热点监控（框架已建，需集成 MediaCrawler） # HEARTBEAT 会在每次检查时执行热点追踪 ``` **配置选项**： - 监控关键词列表（可配置） - 监控频率（每 6 小时） - 热度阈值（互动率超过多少算热门） #### 模块 1.4：竞品监控 **功能说明**：监控特定关键词的新增内容，追踪竞品账号数据 **实现原理**： 1. 定义竞品关键词（如："AI助手"、"自动写作"、"小红书自动发帖"） 2. 定期搜索这些关键词的相关内容 3. 统计新增笔记数量、互动数据 4. 识别快速崛起的竞品账号 **使用方法**： ```bash # HEARTBEAT 会在每次检查时执行竞品监控 # 竞品数据会记录到心跳报告 ``` **配置选项**： - 竞品关键词列表（可配置） - 监控频率（每 6 小时） - 竞品账号列表（可配置） #### 模块 1.5：项目进度 **功能说明**：检查小红书项目待办事项的进度 **实现原理**： 1. 读取 `tasks.json` 中的任务状态 2. 统计待办、进行中、已完成的任务数量 3. 识别超期任务（超过预定截止时间） 4. 生成项目进度报告 **使用方法**： ```bash # HEARTBEAT 会在每次检查时执行项目进度检查 # 查看任务列表 python3 task-manager.py list --todo # 完成任务 python3 task-manager.py done ``` #### 模块 1.6：记忆更新 **功能说明**：每 3 天一次深度整理 MEMORY.md **实现原理**： 1. 检查上次更新时间 2. 如果超过 3 天，提示需要整理 3. 整理内容包括：新增学习笔记、项目进展、技术总结 **使用方法**： ```bash # HEARTBEAT 会在每次检查时检测记忆更新 # 手动触发整理 # 命令待实现 ``` #### 模块 1.7：机会推荐 **功能说明**：发现潜在合作机会、变现渠道、学习资源 **实现原理**： 1. 分析热点追踪和竞品监控的数据 2. 识别潜在的合作伙伴和客户 3. 发现新的变现渠道和商业模式 4. 推荐相关的学习资源和技能提升方向 **使用方法**： ```bash # HEARTBEAT 会在每次检查时生成机会推荐 # 推荐会记录到心跳报告中 ``` ### 3.2 功能 2：任务管理系统 **文件结构**： ``` task-manager.py # 主程序 tasks.json # 任务存储 tasks-archive.json # 归档存储 ``` **核心功能模块**： #### 模块 2.1：任务添加 **功能说明**：添加新任务到任务列表，支持优先级和分类 **实现原理**： 1. 读取 `tasks.json`（如果不存在则创建） 2. 生成新任务的唯一 ID 3. 解析任务参数（标题、优先级、分类、截止日期） 4. 将任务添加到列表，按优先级排序 5. 保存到 `tasks.json` **使用方法**： ```bash # 添加高优先级任务 python3 task-manager.py add "完成周报" --high # 添加普通任务（默认中优先级） python3 task-manager.py add "整理文档" # 添加低优先级任务 python3 task-manager.py add "清理缓存" --low ``` **配置选项**： - 任务优先级：high（高）、normal（中）、low（低） - 任务分类：general（通用）、project（项目）、idea（想法） #### 模块 2.2：任务列表 **功能说明**：列出所有任务，支持按状态和优先级过滤 **实现原理**： 1. 读取 `tasks.json` 中的所有任务 2. 应用过滤条件（按状态、优先级） 3. 按优先级排序（高 > 中 > 低） 4. 格式化输出（使用图标表示状态和优先级） **使用方法**： ```bash # 列出所有任务 python3 task-manager.py list # 只列出待办任务 python3 task-manager.py list --todo # 只列出高优先级任务 python3 task-manager.py list --high ``` #### 模块 2.3：任务完成 **功能说明**：标记任务为已完成，记录完成时间 **实现原理**： 1. 读取 `tasks.json` 2. 找到指定 ID 的任务 3. 更新任务状态为 "done" 4. 记录完成时间 5. 将任务移动到归档文件 `tasks-archive.json` 6. 保存两个文件 **使用方法**： ```bash # 完成任务（ID 为任务编号） python3 task-manager.py done 1 # 查看已归档的任务 cat tasks-archive.json ``` #### 模块 2.4：任务删除 **功能说明**：删除任务并归档，支持恢复 **实现原理**： 1. 读取 `tasks.json` 2. 找到指定 ID 的任务 3. 将任务移动到归档文件 `tasks-archive.json` 4. 从 `tasks.json` 中删除任务 5. 保存两个文件 **使用方法**： ```bash # 删除任务（ID 为任务编号） python3 task-manager.py delete 1 # 查看已归档的任务 cat tasks-archive.json ``` #### 模块 2.5：任务统计 **功能说明**：统计任务数量和完成率 **实现原理**： 1. 读取 `tasks.json` 和 `tasks-archive.json` 2. 统计待办任务数量 3. 统计已完成任务数量 4. 计算完成率 5. 按优先级分类统计 **使用方法**： ```bash # 查看任务统计（输出在任务列表中） python3 task-manager.py list ``` ### 3.3 功能 3：小红书全自动闭环 **文件结构**： ``` xhs-auto-pipeline.py # 自动化流程脚本 xhs-auto-records.json # 发布记录 ``` **核心功能模块**： #### 模块 3.1：热点监控 **功能说明**：监控小红书目标关键词的热度变化，发现潜在爆款话题 **实现原理**： 1. 定义监控关键词列表（AI工具、效率神器、副业搞钱） 2. 使用 MediaCrawler 抓取相关笔记数据 3. 分析笔记数量、点赞、收藏、评论等指标 4. 识别热度上升的趋势 5. 选择最热门的话题作为策略输入 **使用方法**： ```bash # 测试模式（使用模拟数据） python3 xhs-auto-pipeline.py test # 真实模式（需集成实际 API） python3 xhs-auto-pipeline.py ``` **配置选项**： - 标题长度限制（20 字） - 内容风格（教程型、分享型、种草型） - 标签数量和类型 #### 模块 3.4：配图生成 **功能说明**：生成小红书笔记的封面和配图 **实现原理**： 1. 根据内容生成模块的提示词 2. 调用 Grsai API 生成图片 3. 生成封面图（突出标题、吸引点击） 4. 生成配图（展示内容细节） 5. 保存图片到本地 **使用方法**： ```bash # 配图生成是自动化的，会在内容生成后自动执行 # 使用 Grsai API ``` **配置选项**： - 数据收集频率（每 6 小时） - 数据保存格式（JSON） ### 3.4 功能 4：日程集成 **文件结构**： ``` calendar-manager.py # 日程管理脚本 calendar.json # 日程存储 reminders.json # 提醒记录 ``` **核心功能模块**： #### 模块 4.1：添加日程 **功能说明**：添加日程事件（日期、时间、时长、提前提醒） **实现原理**： 1. 读取 `calendar.json` 2. 生成新事件的唯一 ID 3. 解析日程参数 4. 将事件添加到列表 5. 保存到 `calendar.json` **使用方法**： ```bash # 添加日程（日期 + 时间） python3 calendar-manager.py add "客户会议" 2026-02-21 14:00 # 添加全天事件（无时间） python3 calendar-manager.py add "团队建设活动" 2026-02-22 # 自定义时长（分钟） python3 calendar-manager.py add "技术评审" 2026-02-21 16:00 90 ``` **配置选项**： -
