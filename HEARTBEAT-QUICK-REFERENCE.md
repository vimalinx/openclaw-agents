## 📋 当收到 HEARTBEAT 提示时...

### 步骤 1：读取配置
```
读取 HEARTBEAT.md → 了解检查项清单
读取 heartbeat-data.json → 查看上次检查时间
```

### 步骤 2：执行检查（轮换执行）

**早8点检查重点：**
- 📅 日程提醒（未来24小时）
- 📧 邮箱检查（未读重要邮件）
- 📊 项目状态（待办进度）
- 🎯 EvoMap 监控（AI代理机会）

**晚8点检查重点：**
- 🔒 HackerOne 监控（安全悬赏）
- 📌 Pinchtab 监控（技术任务）
- 💰 Algora 监控（开源悬赏）
- 🎓 技能学习进度
- 📊 工具使用统计

### 步骤 3：更新数据
```
更新 heartbeat-data.json：
  - lastChecks 时间戳
  - bountyTracking 新任务
  - skillLearning 学习进度
  - toolUsage 使用次数
```

### 步骤 4：生成报告

按 HEARTBEAT.md 中的报告格式生成结构化报告。

### 步骤 5：决定是否发送

**发送报告的情况：**
- ⏰ 有日程提醒（<2小时）
- 🔥 发现高价值悬赏（>$5K）
- ⚠️ 发现高风险项
- ✅ 完成重要里程碑
- 📊 超过8小时未报告

**只回复 HEARTBEAT_OK 的情况：**
- 🌙 深夜（23:00-08:00）
- 📝 30分钟内刚检查过
- 😐 无新内容
- 🚫 人类正在忙碌
