### 2. 自定义AI代理构建教程

#### 🎓 教程定位
**标题**: 《从零开始构建自定义AI代理：完全初学者指南》

#### 🔧 核心概念

##### 通用AI vs 自定义代理

| 维度 | 通用AI | 自定义代理 |
|------|---------|-----------|
| 工作方式 | 回答问题 | 执行完整工作流 |
| 交互 | 你管理每一步 | 代理自主管理 |
| 一致性 | 每次不同 | 始终遵循一致过程 |

**核心差异**：
- 通用AI：你问"总结会议" → AI给3个要点 → 你问"提取行动项" → AI给列表 → 你要求格式化 → AI改格式
- 自定义代理：你上传会议记录 → 代理自动分析 → 识别决策 → 提取行动项 → 高亮引用 → 生成执行摘要 → 专业格式化 → 询问是否发送
- → **代理管理整个工作流**

##### 自定义代理的4大特征

1. **专业化**: 像某个领域的专家（营销、研究、代码审查、UI设计等）
2. **定义工作流**: 遵循你设计的、一致的多步骤流程
3. **配备工具**: 拥有特定能力（网络搜索、文件读取、截图等）
4. **清晰指导**: 知道你的偏好、约束和质量标准

##### 何时需要自定义代理

问自己：
- ✅ 我是否重复做这个任务？ → 代理可以自动化
- ✅ 这是否需要多个步骤？ → 代理可以遵循流程
- ✅ 我需要一致性质量？ → 代理维护标准
- ✅ 这是否花费大量时间？ → 代理节省时间

##### 实际代理类型

| 代理类型 | 功能 | 用户 |
|---------|------|------|
| 研究助理 | 多源搜索 + 综合分析 + 专业格式 | 学生、分析师 |
| 代码审查 | 分析bug/安全问题 + 最佳实践检查 + 优化建议 | 开发者、技术负责人 |
| 麦肯锡顾问 | 深度研究 + 数据清洗 + 战略洞察 + 执行报告 | 咨询师、业务分析师 |
| 内容创作者 | 保持品牌调性 + 社媒内容生成 + 平台适配 | 营销人员、社媒经理 |

**核心洞察**: 自定义代理不只是回答问题，而是自主执行整个工作流！

### 4. 子代理

#### 🤔 什么是子代理？

- **超专业化专家**: 每个子代理是某个领域的专家
- **按需协作**: 主代理智能决定何时调用
- **专注执行**: 子代理专注完成任务

##### 主代理与子代理交互流程

1. 你给主代理一个任务
2. 它识别哪个子代理需要
3. 将具体子任务自动转交给子代理
4. 子代理执行工作
5. 结果返回给主代理

##### 子代理创建4步

**Step 1**: 识别需求
- 什么任务反复出现？
- 在哪里需要专业化？

**Step 2**: 给子代理定义
- 告诉主代理"何时和为什么"调用这个子代理

**Step 3**: 设计系统提示词
- 步骤流程
- 最佳实践
- 约束和避免事项

**Step 4**: 配置子代理工具
- 只分配完成任务所需的工具

##### 最佳实践

**DO**:
- 给每个子代理清晰、专注的角色
- 提供何时应该被咨询的上下文
- 允许子代理之间必要时通信
- 测试代理之间的交接

**DON'T**:
- 让子代理太宽泛 - 专业化是关键
- 一次添加太多 - 从小开始增长

#### Step 3: 上下文

```markdown
【当前背景】
- 目标平台：小红书（为主），抖音、视频号为辅
- 账号现状：新账号起步期，需要快速冷启动
- 内容定位：AI、科技、效率工具等垂直领域
- 目标用户：20-35岁职场人士、创业者、AI爱好者

【平台特点】
- 小红书：种草+图文为主，重内容质量，算法推荐个性化
- 抖音：短视频为主，重视频质量，算法推荐流量池
- 视频号：长视频为主，重知识密度，算法推荐专业化

【内容策略】
- 80% 原创干货内容，20% 互动内容
- 每周更新 5-7 篇内容
- 内容风格：专业但不失活泼，数据驱动
- 发布时间：小红书 12:00/14:00/18:00/22:00
- 抖音：晚高峰期（18:00-22:00）
- 视频号：周末上午（10:00-12:00）

【竞争对手】
- 主要竞品：AI工具类大号、效率类博主
- 竞品策略：每日 3-5 篇高质量内容，快速更新
- 粉丝规模：10-50 万粉丝
- 差异化优势：更垂直的内容、更深入的分析
```

#### Step 4: 工作流设计

```markdown
【每周工作流】

周一：热点分析 + 内容规划
1. 搜索上周热门话题和趋势
2. 分析竞品爆款内容
3. 制定本周内容规划（5-7篇）
4. 优化发布时间表
5. 生成内容素材（标题、文案、封面图）

周二-周四：内容创作 + 发布
1. 根据规划创作图文/短视频内容
2. 使用AI辅助生成封面图和文案
3. 内容审核和优化
4. 批量或定时发布
5. 监控发布效果
6. 回复用户评论和私信

周五：数据分析 + 优化总结
1. 收集本周所有内容数据
2. 分析内容表现（浏览、点赞、收藏、评论、转发）
3. 分析粉丝增长数据
4. 进行A/B测试结果分析
5. 识别最佳表现内容
6. 总结运营经验和教训
7. 生成下周优化计划

每日：用户互动 + 数据监控
1. 回复评论和私信
2. 监控内容数据变化
3. 记录用户反馈
4. 更新内容策略
```

#### Step 5: 指导原则

```markdown
【优先级】
1. **内容质量第一**: 宁可少发，也要保证每篇内容都是高质量的
2. **数据驱动决策**: 所有运营决策都基于真实数据，不凭感觉
3. **用户价值优先**: 内容必须对用户有价值，不能是垃圾内容或虚假宣传
4. **持续优化**: 不断测试和优化，提升运营效果

【约束】
1. **不发布违规内容**: 不发布违反平台规则的内容
2. **不发布敏感话题**: 不发布政治敏感、色情低俗、争议内容
3. **不发布与定位无关**: 不发布与账号定位无关的内容
4. **不诱导违规互动**: 不诱导刷赞、互粉、引流等违规行为

【质量标准】
- 内容质量高，信息密度大，用户读完有收获
- 封面和标题吸引人但不标题党
- 文案流畅自然，易于理解和接受
- 发布时间准确，符合用户习惯
- 互动及时有效，建立信任关系

【沟通风格】
- 专业但不失活泼，符合自媒体调性
- 数据说话，用数据支撑观点
- 主动建议，提供优化思路
- 语言简洁，直奔主题
- 结构清晰，分章节呈现

【边缘情况处理】
- 如果内容效果不好，及时分析原因并调整策略
- 如果遇到平台限流，调整发布频率或使用备用策略
- 如果发现竞品抄袭，保留证据并备案
- 如果遇到用户负面反馈，及时回复并处理
```
